# Stakeholder Review Playbook

**Owner:** Product Manager | **Cadence:** Weekly / per-milestone  
**Last Updated:** 2025-06-03

---

## Purpose

Keep stakeholders informed, gather input, and maintain alignment on MBO progress.

---

## Review Cadence

| Audience | Frequency | Format | Owner |
|----------|-----------|--------|-------|
| Executive leadership | Monthly | Written + 30min call | All Managers |
| Cross-unit peers | Weekly | Async update | Unit Managers |
| Customers / Beta users | Per feature | Demo + survey | Product Manager + UI/UX |
| Security / Compliance | Per release | Sign-off doc | Security Manager |
| MBO Engine | Continuous | Metrics feed | All Managers |

---

## Weekly Status Template

```markdown
## Weekly Status — [Unit Name] — [Date]

### MBO Progress
- Target: [objective]
- This week: [progress, %]
- Trend: [up / flat / down]

### Shipped
- [Feature A] — [impact]
- [Fix B] — [why it matters]

### In Progress
- [Feature C] — [ETA, blocker?]

### Risks & Asks
- [Risk 1] — Need [help]
- [Risk 2] — Status: [monitoring / mitigated]

### Next Week
- [Top 1-3 priorities]
```

---

## Stakeholder Review Meeting

### Structure (30 min)

| Time | Segment | Owner |
|------|---------|-------|
| 5 min | MBO progress | All Managers |
| 10 min | Demo / walkthrough | Product Manager + Dev |
| 10 min | Decisions needed | Product Manager |
| 5 min | Q&A | All |

### Facilitation Tips

- Send agenda + read-ahead 24h ahead
- One speaker per topic
- End with explicit asks and owners
- Capture decisions in writing

---

## Demo Best Practices

- [ ] Demo working software, not slides
- [ ] Real customer scenario, not contrived data
- [ ] Include "before" comparison
- [ ] Show, don't tell
- [ ] Invite questions live

---

## Decisions Log

Capture every decision in a shared log:

```markdown
## [Date] — Decision

**Context:** [Why this came up]
**Decision:** [What we chose]
**Alternatives considered:** [Other options]
**Trade-offs:** [What we gave up]
**Owner:** [Who owns follow-through]
```

---

## Anti-Patterns

| Avoid | Why |
|-------|-----|
| Status meetings with no decisions | Wastes time |
| Over-promising to look good | Erodes trust |
| Hiding risks | Surprises later |
| Skipping customer demos | Lose feedback signal |
